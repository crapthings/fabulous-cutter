# AGENTS.md

## Purpose
This file defines project-wide rules for contributors and coding agents.

## Rules
1. Use Standard JavaScript Style for all `.js` and `.jsx` code.
2. Keep changes small, focused, and easy to review.
3. Prefer clear, descriptive names over abbreviations.
4. Add or update tests for behavior changes.
5. Do not commit secrets, API keys, or credentials.
6. Update docs when behavior, setup, or commands change.
7. Run lint and tests before opening a PR.

## Workflow
1. Create a focused branch for each change.
2. Make incremental commits with clear messages.
3. Open a PR with a short summary, test notes, and risks.

## Review Checklist
- Code follows project style and lint rules.
- Behavior changes are covered by tests.
- Error handling and edge cases are considered.
- Documentation is updated where needed.
